Sekwencja tworzenia struktury projektu strony internetowej

Tworzenie struktury folderów i plików projektu strony internetowej, która będzie wykorzystywać narzędzia takie jak Gulp, SASS, Git i GitHub, może być skomplikowane, szczególnie dla początkujących programistów. Choć informacje na ten temat są dostępne, często są rozproszone i wymagają dogłębnego poszukiwania, co sprawia, że proces ten staje się czasochłonny. W artykule przedstawiono kompleksowe, praktyczne podejście do zorganizowania projektu od podstaw, co pomoże zminimalizować trudności i usprawnić pracę nad projektem internetowym.

Szczegóły

  • W projekcie wykorzystano zapisy Sass @use i @forward

1. Przygotowanie środowiska

1.1 Zainstaluj Node.js

Upewnij się, że masz zainstalowany Node.js, ponieważ Gulp i Sass są oparte na tym środowisku. Możesz pobrać go ze strony Node.js

1.2 Utwórz nowy folder projektu

Utwórz nowy folder dla swojego projektu i otwórz go w wybranym edytorze kodu, np. Visual Studio Code. Otwórz terminal i przejdź do folderu projektu.

1.3 Zainicjuj projekt npm

W terminalu wpisz npm init -y, aby stworzyć plik package.json z domyślnymi ustawieniami.

2. Instalacja Gulp i Sass

2.1 Zainstaluj Gulp globalnie

npm install --global gulp-cli

2.2 Zainstaluj Gulp i Sass w projekcie

npm install --save-dev gulp gulp-sass sass

3.2 Konfiguracja pliku gulpfile.js

'use strict';

import { src, dest, watch as gulpWatch, task, series } from 'gulp';
import gulpSass from 'gulp-sass';
import * as sass from 'sass';

const sassCompiler = gulpSass(sass);

function buildStyles() {
return src('./src/sass/**/*.scss')
.pipe(sassCompiler().on('error', sassCompiler.logError))
.pipe(dest('./dist/css'));
}

function watchFiles() {
gulpWatch('./src/sass/**/*.scss', buildStyles);
}

task('buildStyles', buildStyles);
task('watch', watchFiles);
task('default', series(buildStyles, watchFiles));

4. Inicjalizacja Git i GitHub

4.1 Zainicjuj repozytorium Git

git init

4.2 Przykładowe ustawienia .gitignore

node_modules/
dist/
projekt.code-workspace

4.3 Dodaj pliki do repozytorium

git add .
git commit -m "Initial commit"

4.4 Utworzenie repozytorium na GitHubie

Utwórz nowe repozytorium na GitHubie i skopiuj jego URL. Połącz lokalne repozytorium z GitHubem

git remote add origin [URL_REPOZYTORIUM]
git push -u origin master
